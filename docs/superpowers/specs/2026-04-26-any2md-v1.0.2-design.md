## 1. Goals and non-goals

### 1.1 Goals

Patch release that closes issue #15 *and* every related quality issue surfaced during deep investigation against four representative real-world inputs (AI Governance arxiv paper, ISO/IEC 27002 standard, COMP4441 academic DOCX, Wikipedia URL). Eleven distinct frontmatter and body-content artifacts identified — all in scope.

### 1.2 Non-goals (deferred to v1.1+)

- LLM-generated abstracts (explicit non-goal in v1.0 design).
- Backend changes (done in v1.0.1).
- Per-format heuristic dispatch (`heuristics/academic.py`, etc.) — over-engineering for a patch.
- Disk-based arxiv response cache.
- Multiple-retry strategies for arxiv.
- New frontmatter validators beyond what already exists.

### 1.3 Locked decisions (from brainstorm Q&A)

| # | Decision | Choice |
|---|---|---|
| Q1 | Heuristic strictness when ambiguous | **C — Profile-gated.** Default `aggressive` = best-effort; `--profile conservative` = fail-safe (leave field empty when in doubt). |
| Q2 | Scope cuts — drop any of the 11? | **A — All 11.** Per user directive to fix everything found. |
| Q3a | New `produced_by` extension field for PDF Creator software junk? | **ii — Yes.** Adds backwards-compatible extension field next to `extracted_via`. |
| Q3b | Author detection scope | **iii — Academic byline + "Authors:" / "By" prefixes + arxiv API lookup.** |
| Q4a | Arxiv lookup default behavior | **iii — Default-on with `--no-arxiv-lookup` opt-out flag.** |
| Q4b | Arxiv response cache | **i — No cache.** |
| Arch | Implementation shape | **Approach 2 — dedicated `heuristics.py` module + 5 new pipeline stages.** |
| Arxiv error handling | User refinement on Section 2 | **Non-blocking warning via existing `add_warnings` channel; conversion proceeds.** |

## 2. Issue catalog (the eleven)

### Category A — Frontmatter heuristic misses (the original #15 trio)

| # | Severity | Symptom | Fix |
|---|---|---|---|
| A1 | HIGH | Authors not extracted from PDF body when `/Author` metadata empty (academic PDFs) | `heuristics.extract_authors` chain: arxiv API → "Authors:"/"By" prefix → academic byline pattern |
| A2 | HIGH | `extract_abstract` picks the byline / cover blurb / TOC line as the abstract | `heuristics.refine_abstract`: skip-list (byline/cover/TOC patterns), prefer `## Abstract` heading body, strip markdown links + HTML entities |
| A3 | MED | `organization` populated with PDF Creator software string ("LaTeX with acmart...", "Adobe InDesign 16.2") | `heuristics.filter_organization`: detect software patterns, return `(organization=None, produced_by=value)`; converter writes `produced_by` to SourceMeta |

### Category B — Universal: HTML entities leak to body

| # | Severity | Symptom | Fix |
|---|---|---|---|
| B1 | HIGH | `&amp;`, `&gt;`, `&lt;` appear in body text from any pipeline that touches HTML internally (Docling, mammoth+markdownify, trafilatura) | New shared cleanup stage **C8 `decode_html_entities`** — applies `html.unescape()` outside fenced code blocks |

### Category C — Title detection misses

| # | Severity | Symptom | Fix |
|---|---|---|---|
| C1 | MED | ISO 27002 title detected as "INTERNATIONAL STANDARD" (cover-page H1, not the actual title) | `heuristics.refine_title` — skip cover-page boilerplate H1s ("INTERNATIONAL STANDARD", "TECHNICAL REPORT", etc.); prefer next H2 |
| C3 | LOW | COMP4441 title concatenates course code + project: "COMP 4441 Final Project Safety Alignment Effectiveness in Large Language Models" | `heuristics.refine_title` — DOCX line-break splitter (conservative: only when explicit delimiter detected) |
| C4 | LOW | Wikipedia title keeps "Wikipedia:" namespace prefix | `heuristics.refine_title` — strip `Wikipedia:` when source_url ends in `*.wikipedia.org` |

### Category D — Body junk

| # | Severity | Symptom | Fix |
|---|---|---|---|
| D1 | MED | Academic PDF body has TOC dumped as a markdown table after the abstract (~75 lines of noise) | New text-lane stage **T7 `dedupe_toc_table`** — detect leading table whose rows look like TOC entries and ≥70% match later H2/H3 headings |
| D2 | MED | Trafilatura sometimes leaves fragments in the body ("This is an", lone "\|") | New text-lane stage **T10 `strip_web_fragments`** — drop trailing-incomplete sentence fragments and orphan punctuation lines |
| D3 | MED | ISO cover page's "Please share your feedback... QR code... Customer Feedback Form Third edition 2022-02 Corrected version 2022-03" leaks into body | New text-lane stage **T8 `strip_cover_artifacts`** — pattern-match leading lines containing cover-page keywords and version stamps |

### Category E — Redundant text

| # | Severity | Symptom | Fix |
|---|---|---|---|
| E1 | LOW | Academic PDFs have an "Author's Contact Information:" line duplicating the byline | New text-lane stage **T9 `strip_repeated_byline`** — drop lines matching the contact-info pattern |

## 3. Architecture

### 3.1 High-level flow

```
                  any2md v1.0.2 (changes from v1.0.1)
                  ────────────────────────────────────

   pdf.py / docx.py / html.py / txt.py
              │
              │  v1.0.2: SourceMeta gains produced_by: str | None
              │  PDF/DOCX populate it from PDF Creator / DOCX Application field
              │  via heuristics.filter_organization()
              │
              ▼
       pipeline.run(text, lane, options)
              │
              │  STRUCTURED lane:  S1-S4 (unchanged)
              │  TEXT lane:        T1-T6 (unchanged)
              │                  + T7 dedupe_toc_table        ◀ NEW (D1)
              │                  + T8 strip_cover_artifacts   ◀ NEW (D3)
              │                  + T9 strip_repeated_byline   ◀ NEW (E1)
              │                  + T10 strip_web_fragments    ◀ NEW (D2)
              │  SHARED:          C1-C7 (unchanged)
              │                  + C8 decode_html_entities    ◀ NEW (B1, universal)
              ▼
        cleaned_md + warnings
              │
              ▼
   frontmatter.compose(body, meta, options, overrides)
              │
              │  v1.0.2: compose() consults heuristics.py before YAML emission:
              │    - title       = heuristics.refine_title(...)
              │    - abstract    = heuristics.refine_abstract(...)
              │    - authors     = heuristics.extract_authors(...) — may call arxiv
              │    - produced_by = (already populated by converter)
              │
              ▼
       YAML + body output
```

### 3.3 Module boundary principle

`heuristics.py` is a leaf module of pure functions (one network call exception: `arxiv_lookup`). `frontmatter.compose()` calls into it; nothing in `heuristics.py` imports from converters or pipeline modules — the `add_warnings` import inside `arxiv_lookup` is the lone exception, lazy-imported inside the function body to avoid an import cycle.

Pipeline stages (T7–T10, C8) are format-agnostic: they act on text patterns visible in *any* lane-text input regardless of which converter produced it. They live in the existing `pipeline/text.py` and `pipeline/cleanup.py` files alongside their siblings.

## 4. `heuristics.py` module contract

### 4.1 Public API

```python
"""any2md/heuristics.py

Field-derivation refinements that go beyond what raw extractors give us.
Every function is pure (input → output, no side effects) except
arxiv_lookup which makes one network call and emits a warning on failure.

Called from frontmatter.compose() and converter modules to refine
candidate values before YAML emission.
"""

from __future__ import annotations

import re
from typing import NamedTuple

from any2md.pipeline import Profile  # type alias


class OrgFilterResult(NamedTuple):
    organization: str | None
    produced_by: str | None


def filter_organization(creator_value: str | None) -> OrgFilterResult:
    """Distinguish real organization names from PDF Creator software junk.

    - When `creator_value` matches a known software pattern, returns
      OrgFilterResult(None, creator_value).
    - Otherwise returns OrgFilterResult(creator_value, None).
    - Empty/None input returns OrgFilterResult(None, None).
    """


def refine_title(
    candidate: str,
    body: str,
    *,
    source_url: str | None = None,
    profile: Profile = "aggressive",
) -> str:
    """Replace candidate when it looks like cover-page boilerplate.

    Behaviors:
      - If candidate (case-insensitive, stripped) matches a known
        cover-page-boilerplate H1 ("INTERNATIONAL STANDARD",
        "TECHNICAL REPORT", "WHITE PAPER", etc.), prefer the first H2
        in the body as the title.
      - For source_url ending in *.wikipedia.org, strip a leading
        "Wikipedia:" or "WP:" namespace prefix.
      - DOCX line-broken H1 (aggressive only): when candidate contains
        an explicit delimiter (" - ", colon-and-space at < 30% mark)
        AND the segment before the delimiter looks like a course code or
        document type prefix (e.g., "COMP 4441 Final Project"), drop the
        prefix.

    Conservative profile: only the cover-page-boilerplate skip and the
    Wikipedia namespace strip. DOCX line-break refinement stays off.
    """


def refine_abstract(
    candidate: str | None,
    body: str,
    *,
    profile: Profile = "aggressive",
) -> str | None:
    """Replace candidate when it looks like a byline / cover blurb / TOC.

    Selection (in order):
      1. Body section under "## Abstract" / "## ABSTRACT" / "## Summary"
         (case-insensitive H2/H3) — take its first paragraph >= 80 chars.
      2. First non-skip paragraph >= 80 chars after H1 (current
         heuristic with skip-list applied).
      3. None.

    Skip patterns (move to the next paragraph if these match):
      - Author byline: starts with caps-heavy text, >= 3 commas, >= 1
        digit (affiliation marker).
      - Cover-page blurb: contains any of "feedback", "qr code",
        "scan the", "customer feedback form" (case-insensitive).
      - TOC line: contains "....\\d+" or "page \\d+" / "Page \\d+".

    Post-selection cleanup (always applied):
      - Strip inline markdown link syntax: [text](url) → text.
      - Decode HTML entities (html.unescape).
      - Truncate to <= 400 chars at last sentence boundary.

    Conservative profile: skip-list still active, but if no qualifying
    paragraph found returns None instead of falling through to a
    skipped paragraph. (Default aggressive returns the first non-skip
    even if it's a fallback.)
    """


def extract_authors(
    body: str,
    title_hint: str | None,
    arxiv_id: str | None = None,
    *,
    arxiv_lookup_enabled: bool = True,
    profile: Profile = "aggressive",
) -> list[str]:
    """Extract authors from body, optionally enriched with arxiv API.

    Detection chain (first match wins):
      1. arxiv_lookup(arxiv_id) result (when arxiv_id is set and
         arxiv_lookup_enabled is True; warns on failure).
      2. "Authors: <list>" or "Author: <name>" prefix line.
      3. "By <list>" prefix line near top of body (within first 20
         lines after H1).
      4. Academic byline pattern: caps-or-titlecase names, optional
         digit-affiliation markers, comma-separated, on the line(s)
         immediately after H1. Splits on commas-and-digit-groups.

    Returns deduplicated, order-preserving author list (max 20).
    Conservative profile: only patterns 1, 2, 3 (skips byline-pattern
    inference); returns [] when none match.
    """


def is_arxiv_filename(name: str) -> str | None:
    """Return arxiv ID if name matches arxiv pattern, else None.

    Accepts both new (\\d{4}.\\d{5}) and legacy (\\d{4}.\\d{4}) formats.
    Strips trailing "v\\d+" version qualifier.

    Examples:
      "2501.17755v1.pdf"    → "2501.17755"
      "1706.03762.pdf"      → "1706.03762"
      "report.pdf"          → None
    """


def arxiv_lookup(arxiv_id: str, *, timeout: float = 5.0) -> dict | None:
    """Fetch metadata from the public arxiv API.

    On success returns:
      {
        "title": str,
        "authors": list[str],   # publication order
        "abstract": str,
        "date": str             # ISO YYYY-MM-DD
      }

    On any failure (SSRF block, network timeout, HTTP non-200, XML
    parse error, schema mismatch), emits a non-blocking warning via
    the pipeline's existing `add_warnings` channel and returns None.
    Conversion never fails because of arxiv unreachability.

    Endpoint: https://export.arxiv.org/api/query?id_list={arxiv_id}
    SSRF-guarded same as html.py (validate IPs against private/
    reserved/loopback). Timeout: 5s default. Single attempt, no retry.
    """
```

# PDF Creator strings that mean "software", not "organization"
_SOFTWARE_CREATORS_RE = re.compile(

# "Authors:" / "Author:" / "By" prefixes
_AUTHORS_PREFIX_RE = re.compile(

    re.IGNORECASE | re.MULTILINE,

# arxiv ID pattern in filename
_ARXIV_FILENAME_RE = re.compile(

### 4.3 Implementation notes

- `arxiv_lookup` uses `urllib.request` (stdlib), not a new dependency. The Atom XML response is parsed with `xml.etree.ElementTree`. Atom namespaces required.
- SSRF guard reuses `_validate_url_host` from `any2md/converters/html.py` (lazy-imported to avoid circular dep).
- Author dedup uses case-insensitive comparison of normalized names (collapse whitespace, strip punctuation).
- `extract_authors` cap at 20 prevents pathological output if a heuristic fires on the wrong block.

## 5. New pipeline stages

### 5.1 Shared cleanup C8 — `decode_html_entities`

**File:** `any2md/pipeline/cleanup.py`. Position: after C5 `collapse_whitespace`, before C6 `strip_footnote_markers`. Universal — runs on every output.

**Behavior:**
- Use `html.unescape()` from stdlib to decode named (`&amp;`, `&gt;`) and numeric (`&#x2014;`, `&#8212;`) entities.
- **Preserve fenced code blocks** (between ` ``` ` markers): track fence state, skip lines inside fences.
- Inline code (single backticks) is NOT skipped — most documents don't have HTML entities inside inline code, and the false-positive rate is acceptable.

**Rationale:** Docling, mammoth+markdownify, and trafilatura all run text through HTML at some point. Entities leak universally. Fixing in shared cleanup catches every path.

### 5.2 Text-lane T7 — `dedupe_toc_table`

**File:** `any2md/pipeline/text.py`. Runs after T4 `dedupe_toc_block` (which handles text-formatted TOCs); T7 handles table-formatted TOCs that T4 misses.

**Behavior:**
- Find tables (consecutive `|`-prefixed lines) in the first 30% of the document.
- For each candidate table, extract apparent title-text from cells (skip integer-only cells which are page numbers).
- If ≥ 70% of extracted titles match later H2/H3 heading text (case-insensitive contains-match), drop the table.

**Profile gating:** aggressive and maximum only.

### 5.3 Text-lane T8 — `strip_cover_artifacts`

**File:** `any2md/pipeline/text.py`. Runs early in text lane (after T1 `repair_line_wraps`, before T3 `dedupe_paragraphs`).

**Behavior:**
- Operate only on the first ~30 lines of the document (after the H1 and any byline).
- Drop a line if it contains any of: "QR code", "scan the", "customer feedback form" (case-insensitive); OR matches `r"^(?:Third|Fourth|Fifth|...)\s+edition\s+\d{4}-\d{2}$"`; OR matches `r"^Corrected version \d{4}-\d{2}$"`.
- Stop at the first H2 (`## ...`) — beyond that, body content is fair game.

**Profile gating:** aggressive and maximum only.

### 5.4 Text-lane T9 — `strip_repeated_byline`

**File:** `any2md/pipeline/text.py`. Runs after T2 `dehyphenate`, before T3 `dedupe_paragraphs`.

**Behavior:**
- Match lines starting with "Author's Contact Information:" / "Authors' Contact Information:" / "Contact:" (case-insensitive) → drop the entire line plus subsequent indented continuations.
- Match lines containing "@" email pattern preceded by name list (looks like author footer) → drop only when within first 50 lines.

**Profile gating:** aggressive and maximum only.

### 5.5 Text-lane T10 — `strip_web_fragments`

**File:** `any2md/pipeline/text.py`. Runs after T5 `strip_running_headers_footers`, before T6 `restore_lists_and_code`.

**Behavior:**
- Drop lines that are obvious extraction fragments:
  - Lines containing only `|` or `>`
  - Lines that are an incomplete sentence (≤ 25 chars, ends without terminal punctuation, not followed by a continuation paragraph) — applied only when the preceding line ended in terminal punctuation (so we know it's not a wrap).
- Profile-gated: aggressive and maximum only.

**Conservatism:** false-positive risk is real here (could drop legitimate short lines like "Contents" or "Note:"). To minimize: require the line be < 25 chars AND surrounded by blank lines AND not match common short-heading patterns.

### 5.6 Profile gating summary

| Stage | conservative | aggressive (default) | maximum |
|---|---|---|---|
| C8 decode_html_entities | ✓ | ✓ | ✓ |
| T7 dedupe_toc_table | – | ✓ | ✓ |
| T8 strip_cover_artifacts | – | ✓ | ✓ |
| T9 strip_repeated_byline | – | ✓ | ✓ |
| T10 strip_web_fragments | – | ✓ | ✓ |

C8 runs at all profiles because HTML entities are universally wrong in the body — there's no legitimate use case for keeping them un-decoded for a human or RAG reader.

## 6. SourceMeta extension

```python
@dataclass
class SourceMeta:
    title_hint: str | None
    authors: list[str]
    organization: str | None
    produced_by: str | None         # NEW in v1.0.2 — software/tool that
                                    # produced the source (PDF Creator field
                                    # for PDFs, Application field for DOCX,
                                    # User-Agent for URLs)
    date: str | None
    keywords: list[str]
    pages: int | None
    word_count: int | None
    source_file: str | None
    source_url: str | None
    doc_type: Literal["pdf", "docx", "html", "txt"]
    extracted_via: Literal[...]
    lane: Lane
```

### 6.1 Frontmatter emission

`produced_by` is emitted as an any2md extension field, alongside `extracted_via`, **after** `type` and **before** `pages`/`word_count`:

```yaml
---
title: "AI Governance through Markets"
…
type: "pdf"
extracted_via: "docling"
produced_by: "LaTeX with acmart 2024/08/25 v2.09 …"   # NEW field
pages: 36
---
```

When `produced_by` is None, the field is omitted (matches the convention for other optional extension fields).

### 6.2 Converter integration

- **`converters/pdf.py`**: `_parse_pdf_metadata` calls `heuristics.filter_organization(meta.get("creator"))` — splits the result into `organization` and `produced_by` fields of the returned dict.
- **`converters/docx.py`**: `_read_docx_metadata` reads `docProps/app.xml` — currently only populates `organization` from `Company`. Now also reads `Application` from app.xml and routes through `filter_organization`. The `Company` value (real organization) takes priority over `Application` (software).
- **`converters/html.py`**: trafilatura's `sitename` is generally a real domain or site name (not software). `filter_organization` is *not* applied for HTML — `produced_by` stays None. (A future enhancement could populate from the `User-Agent: Generated By` HTTP header pattern, but it's rare.)

## 7. CLI surface change

### 7.1 New flag

```
--no-arxiv-lookup
    Disable the arxiv API lookup that enriches frontmatter (authors,
    abstract, date) for PDFs whose filename matches the arxiv ID pattern
    (\d{4}.\d{4,5}). Use this in airgapped environments or when you don't
    want any2md to make outbound network calls. Default: arxiv lookup is
    enabled.
```

### 7.2 PipelineOptions extension

```python
@dataclass(frozen=True)
class PipelineOptions:
    profile: Profile = "aggressive"
    ocr_figures: bool = False
    save_images: bool = False
    strip_links: bool = False
    strict: bool = False
    high_fidelity: bool = False
    auto_id: bool = False
    auto_id_prefix: str = "LOCAL"
    auto_id_type_code: str = "DOC"
    backend: Backend | None = None
    frontmatter_overrides: dict | None = None
    arxiv_lookup: bool = True       # NEW in v1.0.2
```

### 7.3 No other CLI surface changes

Everything else in v1.0.2 is internal. Existing `--profile {conservative,aggressive,maximum}` already gates the new heuristic strictness (Q1=C); no new flag for that.
