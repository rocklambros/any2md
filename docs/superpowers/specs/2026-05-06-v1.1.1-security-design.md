## 1. Background

Comprehensive security scan of `feat/v1.1.0-persistent-converter` (commit `67e838d`) using bandit, snyk-code, gitleaks, pip-audit, trivy, and a manual Claude security-engineer review surfaced 10 actionable findings. v1.1.0 is already on PyPI; this design covers the v1.1.1 patch release that addresses all 10. Five rounds of adversarial review (security architecture → methodology → operations → code review → synthesis attack) refined the implementation, ordering, validation, and rollout. This document is the converged plan.

## 2. Threat model (inline)

`any2md` is an operator-trusted CLI distributed via PyPI for desktop and CI use.

**Trusted:** the operator, their filesystem, their `--output-dir`, and the Python interpreter they invoke. CLI flags and `--meta key=val` arguments are operator-controlled.

**Untrusted:**
- URLs passed to `any2md` (may resolve to internal IPs, redirect, or carry credentials in userinfo/query).
- Documents (PDF/DOCX/HTML) passed for conversion (may embed crafted images, control characters, malicious metadata).
- `.any2md.toml` config files discovered via cwd-walking (an attacker who can drop a file in any ancestor directory can inject frontmatter overrides).
- HTTP responses from URLs (may carry hostile redirects, oversized bodies, control characters in headers).

**Out of scope:** multi-tenant service hosting; running `any2md` against attacker-supplied output directories; supply-chain compromise of pinned dependencies; protection against an operator who has chosen `--output-dir` pointing at a shared/symlinked directory.

This statement also lands in `SECURITY.md` (Commit 12).

### Commit 2 — URL credential + query-param scrub (Fix 5)

**Files:**
- Modified: `any2md/utils.py` — add `scrub_url_credentials`.
- Modified: `any2md/converters/html.py` — call scrubber before persisting `source_url` (line ~185) and before any error-print interpolating the URL (line ~213).
- Modified: `any2md/heuristics.py:556` — same scrub before `_warn` interpolating arxiv URL.
- Modified: `any2md/utils.py:98` (`url_to_filename`) — derive filename from `parsed.hostname + parsed.path`, NOT `netloc` (which carries userinfo).

**Behavior:**
```python
_SECRET_PARAM_RE = re.compile(
    r"^(token|api[_-]?key|access[_-]?token|bearer|signature|sig|password|secret|aws[_-]?session[_-]?token)$",
    re.IGNORECASE,
)

def scrub_url_credentials(url: str) -> tuple[str, list[str]]:
    """Strip userinfo and sensitive query params from a URL.

    Returns (scrubbed_url, warnings). Each warning names the kind that
    was stripped (e.g., "credentials", or a query-param key) but never
    echoes the value.
    """
    parsed = urllib.parse.urlsplit(url)
    warnings = []

    # Userinfo
    if parsed.username or parsed.password:
        warnings.append("credentials")
        host = parsed.hostname or ""
        # IPv6 must round-trip with brackets; urlsplit returns hostname WITHOUT brackets
        if ":" in host:
            host = f"[{host}]"
        netloc = host + (f":{parsed.port}" if parsed.port else "")
        parsed = parsed._replace(netloc=netloc)

    # Query params
    pairs = urllib.parse.parse_qsl(parsed.query, keep_blank_values=True)
    kept = []
    for k, v in pairs:
        if _SECRET_PARAM_RE.match(k):
            warnings.append(k)
        else:
            kept.append((k, v))
    parsed = parsed._replace(query=urllib.parse.urlencode(kept))

    return urllib.parse.urlunsplit(parsed), warnings
```

Caller pattern (in `convert_url`, error prints, etc.):
```python
scrubbed, warns = scrub_url_credentials(url)
for w in warns:
    if w == "credentials":
        _logging.warn("stripped credentials from URL")
    else:
        _logging.warn(f"stripped sensitive query parameter '{w}' from URL")
url = scrubbed  # use scrubbed everywhere downstream
```

**Test:** `tests/unit/test_url_creds_scrubbed.py`
- `https://alice:s3cret@example.com/blog/post` → `https://example.com/blog/post`; `alice`/`s3cret` never appear in output.
- `https://example.com/path?api_key=abc&q=hi` → `https://example.com/path?q=hi`; `abc` never echoed; warning names only `api_key`.
- IPv6 round-trip: `https://user:pw@[::1]:8080/x` → `https://[::1]:8080/x` (brackets preserved).
- `https://user:pw@example.com/?Token=xyz&aws_session_token=stf` → both `credentials` and the two param keys named in warnings; URL emitted contains neither value.
- `url_to_filename("https://user:pw@example.com/blog/post")` → `example_com_blog_post.md` (no `user_pw`).
- Existing URLs without userinfo or matching params round-trip identically.

### Commit 8 — SSRF: pinned-IP connector + ProxyHandler({}) + is_global allowlist + UA refresh (Fix 1 + Fix 3 + OPS-8)

**Combined commit.** Three squashed TDD cycles. Commit message documents the pattern.

**File:** `any2md/_http.py` (substantial rewrite, behavior-preserving for the arxiv-lookup callsite at `heuristics.py:554`).

**Behavior:**

1. **Resolve once per hop.** New helper:
   ```python
   def _resolve_and_validate(hostname: str) -> tuple[list[str], str | None]:
       try:
           infos = socket.getaddrinfo(hostname, None, type=socket.SOCK_STREAM)
       except socket.gaierror:
           return [], f"cannot resolve host: {hostname}"
       ips = []
       for *_, sockaddr in infos:
           try:
               addr = ipaddress.ip_address(sockaddr[0])
           except ValueError:
               continue
           if not (addr.is_global and not addr.is_multicast and not addr.is_unspecified):
               return [], f"URL resolves to disallowed address: {addr}"
           ips.append(sockaddr[0])
       if not ips:
           return [], f"no usable address for {hostname}"
       return ips, None
   ```

2. **Pinned connections.**
   ```python
   class _PinnedHTTPSConnection(http.client.HTTPSConnection):
       def __init__(self, host, *args, ip: str, **kwargs):
           super().__init__(host, *args, **kwargs)
           self._pinned_ip = ip

       def connect(self):
           sock = socket.create_connection(
               (self._pinned_ip, self.port), timeout=self.timeout
           )
           if self._tunnel_host:
               self.sock = sock
               self._tunnel()
           # SNI + cert validation must use the original host, not the IP
           self.sock = self._context.wrap_socket(sock, server_hostname=self.host)
   ```
   Mirror `_PinnedHTTPConnection(http.client.HTTPConnection)` for plain HTTP.

3. **Pinned handlers + opener.**
   ```python
   class _PinnedHTTPSHandler(urllib.request.HTTPSHandler):
       def __init__(self, ip: str):
           super().__init__()
           self._ip = ip
       def https_open(self, req):
           return self.do_open(
               lambda host, **kw: _PinnedHTTPSConnection(host, ip=self._ip, **kw),
               req,
           )
   # Mirror _PinnedHTTPHandler.

   def _build_pinned_opener(ip: str, scheme: str):
       handlers = [_NoFollowRedirect(), urllib.request.ProxyHandler({})]
       handlers.append(_PinnedHTTPSHandler(ip) if scheme == "https" else _PinnedHTTPHandler(ip))
       return urllib.request.build_opener(*handlers)
   ```
   `ProxyHandler({})` MUST be present to suppress `HTTP_PROXY`/`HTTPS_PROXY` env-derived proxies that would bypass IP pinning.

4. **`safe_fetch` rewrite.** Per hop: parse URL → resolve and validate → pick first IP from validated list → build pinned opener for that IP → open. On HTTP redirect (301/302/303/307/308), re-validate the new URL on the next iteration. Existing 20 MB body cap, 15 s timeout, 3-hop limit, and `_NoFollowRedirect` retained.

5. **UA refresh.**
   ```python
   from any2md import __version__
   _USER_AGENT = f"any2md/{__version__}"
   ```
   (Avoids the stale `1.0.6` literal.)

6. **`validate_url`** — keep the public function name; it now delegates to `_resolve_and_validate` and discards the IP list (callers that only need yes/no validation still work).

**Locked error strings (used in tests):**

## 5. Validation gate

Implementation may not merge to `main` until all of:

1. `pytest tests/` — all green, including the 10 new TDD pairs and the locked-error-string manifest.
2. `ruff check . && ruff format --check .` — clean.
3. **CodeQL** workflow run on the PR (public-repo code scanning; verified working without GHAS) — zero new `error`/`warning` alerts compared to the `main` baseline.
4. `pip-audit -r .devcontainer/requirements.lock` (or `pip-audit` on a venv with base deps) — 0 findings.
5. `pip-audit` on a venv with `any2md[high-fidelity]` installed — no NEW HIGH/CRITICAL versus the v1.1.0 baseline (record baseline at PR-open time; expectations follow the SECURITY.md dep-drift policy if drift occurs).
6. bandit / snyk-code / gitleaks / trivy — informational only; not gating (proven blind on this codebase, 1/10 hit-rate).
7. PR open through ≥1 full green CI cycle AND no force-pushes after the final review pass.
8. Manual audit: 17 stderr `print()` sites confirmed routed through `_logging.warn`/`fail` (run `grep -rn "print(.*file=sys.stderr" any2md/` and verify each match either uses `_logging` helpers or has a `# log-safe: literal` comment).

## 7. Risks & rollback

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| SSRF fix breaks normal HTTPS (cert validation regression) | medium | high | TDD asserts `server_hostname` is passed correctly; rc-cycle smoke against a real public URL |
| Stricter IP allowlist breaks a legit user's workflow | low | medium | CHANGELOG breaking note; `curl > file && any2md file` workaround documented |
| Frontmatter key validator rejects an in-the-wild `.any2md.toml` | low | medium | Validator runs against the project's own fixture configs as a CI lint; CHANGELOG breaking note |
| Log sanitizer accidentally swallows useful error context | low | low | `_sanitize_log_text` preserves `\n`/`\r`; `safe_oneline` strips them only for single-line interpolation |
| TestPyPI dep resolution fails for `[high-fidelity]` | low | medium | Single-pass resolver via `--extra-index-url` validated with v1.1.0 prior cycle |
| `gh release create` triggers workflow but TestPyPI no-ops | low | medium | `gh api ... /git/refs/tags/...` verifies tag visibility before release; workflow logs reviewed; `pip index versions any2md --index-url https://test.pypi.org/simple/` confirms |
| Tag-version mismatch breaks `validate` job | low | high | Validate job already fails fast; double-check `__version__` before tagging |
| User's local `pip.conf` makes smoke results irreproducible | medium | low | Documented caveat in smoke section |
| Self-review on `pypi` GitHub Environment is a rubber stamp on solo repo | n/a | medium | Acknowledged limitation; gate strength is the validation gate (§5), not the env review |

**Rollback procedures:**

- **Mid-implementation, pre-merge:** `git reset` the offending commit; re-run from previous green. No external impact.
- **Post-PR-merge, pre-tag:** revert the merge commit on `main`; tag never created so no PyPI footprint.
- **Post-rc, pre-final:** TestPyPI release is throwaway. Don't promote; cut a new rc. Same-version overwrite to TestPyPI is permitted only on a mid-flight failure; otherwise increment to `1.1.1-rc2`.
- **PyPI mid-flight failure:** check workflow logs. If wheel was uploaded successfully, `pip index versions any2md` shows it; cannot re-upload same version. Bump to `1.1.1.post1` or `1.1.2`.
- **Post-PyPI publish regression:**
  1. `pip yank any2md==1.1.1 --reason="<short>"` — yank is metadata only; existing pinned installs unaffected.
  2. CHANGELOG entry for v1.1.1 gets a `**[YANKED]**` marker appended.
  3. Open a revert PR; merge; cut **v1.1.2** (yank ≠ revert; both required).
  4. Decision tree for severity:
     - Security-critical regression → yank immediately + ship v1.1.2 within hours.
     - Functional regression (non-security) → ship v1.1.2 within ~24h, no yank.
     - Edge-case-only regression → document workaround; fix in v1.1.3.

## 8. Appendix A — Adversarial review summary (5 rounds)

5 rounds of adversarial review (security-engineer, root-cause-analyst, devops-architect, code-reviewer, security-engineer) were run. R4 declared convergence (findings shifted from "wrong results" to "implementer confusion"); R5 confirmed convergence. Three load-bearing wrong premises in the original design were corrected:

1. **v1.1.0 is already on PyPI** (commit `43caf0e` on `origin/main`, tag `v1.1.0`). The original two-phase plan ("ship v1.1.0 first") collapsed into a single v1.1.1 release.
2. **Severity-ordered commits with SSRF first** was suboptimal because Fix 5 and Fix 9 depend on the `_logging` module from Fix 10. New order: Fix 10 first; Fix 1/3 (combined) last.
3. **SSRF fix as originally specified did not block proxy-env bypass.** `ProxyHandler({})` must be passed explicitly into `build_opener`.

Three load-bearing technical bugs were corrected:

4. **URL scrub helper corrupted IPv6 URLs** (naïve `urlsplit→_replace(netloc=hostname+port)→urlunsplit` strips `[::1]` brackets). Fix reconstructs brackets when `:` in hostname.
5. **Fix 6 only flipped one workflow job's `attestations: false`.** Both `publish-testpypi` AND `publish-pypi` jobs are flipped.
6. **Smoke install used a fragile split procedure.** Replaced with single-pass resolver (`--index-url https://test.pypi.org/simple/ --extra-index-url https://pypi.org/simple/`).

Rejected overreach (with rationale):

- `THREATMODEL.md` document → inline section in this design + SECURITY.md (R2, R4: theatre for solo CLI repo).
- NFC/case-fold YAML reserved-key concern → not exploitable; reserved keys are 6 ASCII snake_case strings (R2 verified, R4 confirmed).
- `atomic_write_text` parent-directory hardening → operator-trusted threat model; deferred to v1.2 (R2, R4 E10).
- "External reviewer" gate → unimplementable on solo repo; replaced with measurable "≥1 full green CI cycle + no force-pushes" gate (R3).
- `pytest-httpserver` swap for SSRF tests → fights the new validator (loopback rejected); bespoke `_FakeOpener` retained (R4 E6).
- AST-walker test for log sweep → too clever for 17 sites; manual audit suffices (R4).
- Anaconda+docling pessimism → not a release blocker; documented caveat sufficient (R4).
- Missing 50 MB byte cap on URL fetch → INVALID; existing `_http.py:22,102-104` already caps at 20 MB (R5 verified).

Two R5 findings were rejected after empirical verification:

- R5 byte-cap finding: existing 20 MB cap is sufficient.
- R5 docling pillow constraint conflict: docling 2.92.0 declares no pillow constraint on PyPI; the original concern was based on a misread of `docling-slim`.
